  1. I start working locally on a brand new project
  2. I go to GitHub and create a new repository for my project
  3. Confused, start Google-ing how to connect the two of them

How do I connect them?

Here’s what you need to do to connect your local git repo with your remote github repo:

cd /my/local/project/path/
git init
git remote add --track master origin git@github.com:jorgepedret/foo.git
  1. CD into your local project directory
  2. Run git init, if you haven’t ran it yet
  3. Connect the remote repo with your local repo

Done!

Now you can commit your changes, pull rebase and then push to your remote repository

git add .
git commit -m "Initial commit"
git pull --rebase origin master
git push origin master

Another way to do it

Here’s another way to connect your local repo with a remote github repo:

  1. Rename your local project directory
  2. Clone your GitHub repo specifying the original directory
  3. Move the original content to the new repository
  4. Delete the temporary directory created in line 1
mv my_project my_project_temp
git clone https://github.com/jorgepedret/foo.git my_project
cp -r ./my_project_temp/. my_project
rm -r ./my_project_temp

What’s the correct way to do it?

The correct way to do it is to start your GitHub repository before starting to code. That way you can simply use git clone and then just start working on it.

That’s it! Thanks to Caius Theory and Ciarán Walsh for helping me find a solution for this problem.

He loved every single part of it, except that it was all in English and he wanted to see it in Spanish. Que problem-o.

Using a different language for WordPress’ admin, was something I’ve always wanted to learn, so I started looking around and I couldn’t believe how easy it was…

1. Download the language files

Go to this url http://codex.wordpress.org/WordPress_in_Your_Language and search for the language that you’re looking for:

  1. Find the language that you’re looking for (i.e.: es_ES)
  2. Download the whole WordPress package for that language
  3. Uncompress the downloaded .zip file
  4. All the language files will be under the /wp-content/languages/ directory

2.Move the languages files to your current WordPress installation

Copy the language files from the previously uncompressed package (from /wp-content/languages/) and move them to /wp-content/languages/ in your current WordPress installation.

3. Change the wp-config.php file

Go to the /wp-config.php file in your current WordPress installation and change the WPLANG constant to your language.

This is what mine looks like:

define('WPLANG', 'es_ES');

That’s it!

Reload your admin panel and it should now be translated to your own language (or your clients’).

While researching this stuff, I also found an amazing plugin to manually translate WordPress templates that use the __() and _e() l10n’s functions. If you don’t know what I’m talking about check this out http://codex.wordpress.org/I18n_for_WordPress_Developers. Add the placeholder option to the editor

/* In the functions.php file */
/* Add a custom field to the field editor (See editor screenshot) */
add_action("gform_field_standard_settings", "my_standard_settings", 10, 2);
function my_standard_settings($position, $form_id){
	// Create settings on position 25 (right after Field Label)
	if($position == 25){
		?>
		<li class="admin_label_setting field_setting" style="display: list-item; ">
			<label for="field_placeholder">Placeholder Text
				<!-- Tooltip to help users understand what this field does -->
				<a href="javascript:void(0);" class="tooltip tooltip_form_field_placeholder" tooltip="&lt;h6&gt;Placeholder&lt;/h6&gt;Enter the placeholder/default text for this field.">(?)</a>
			</label>
			<input type="text" id="field_placeholder" class="fieldwidth-3" size="35" onkeyup="SetFieldProperty('placeholder', this.value);">
		</li>
		<?php
	}
}

/* Now we execute some javascript technicalitites for the field to load correctly */
add_action("gform_editor_js", "my_gform_editor_js");
function my_gform_editor_js(){
	?>
	<script>
		//binding to the load field settings event to initialize the checkbox
		jQuery(document).bind("gform_load_field_settings", function(event, field, form){
			jQuery("#field_placeholder").val(field["placeholder"]);
		});
	</script>
	<?php
}

Use jQuery to inject the placeholder value to the fields

/* In the functions.php file */
/* We use jQuery to read the placeholder value and inject it to its field */
add_action('gform_enqueue_scripts',"my_gform_enqueue_scripts", 10, 2);
function my_gform_enqueue_scripts($form, $is_ajax=false){
	?>
	<script>
	jQuery(function(){
		<?php
		/* Go through each one of the form fields */
		foreach($form['fields'] as $i=>$field){
			/* Check if the field has an assigned placeholder */
			if(isset($field['placeholder']) && !empty($field['placeholder'])){
				/* If a placeholder text exists, inject it as a new property to the field using jQuery */
				?>
				jQuery('#input_<?php echo $form['id']?>_<?php echo $field['id']?>').attr('placeholder','<?php echo $field['placeholder']?>');
				<?php
			}
		}
		?>
	});
	</script>
	<?php
}

That’s it! take it for a spin and let me know how it goes.

Here are 3 tools that I’ve been using lately to show placeholder images in the sites I design: Lorempixum, Flickholdr & Placehold.it

Lorempixum

http://lorempixum.com/

It was recently launched , but I’ve found it to be one of the best tools so far. You can specify the size, the category and if you want it colored or black & white.

Here are some examples of how to use it, simply put this in the src parameter of your img tag:


http://lorempixum.com/400/200


http://lorempixum.com/g/400/200


http://lorempixum.com/400/200/sports


http://lorempixum.com/400/200/sports/1

Flickholdr

http://flickholdr.com/

Works the same way as Lorempixum, but looks for the images in Flickr. It makes sure that the images’ copyright allow you to use them anywhere.

The thing that I like about Flickholdr that Lorempixum doesn’t have, is that you can choose your own tags, instead of using fixed tags.

I had a couple of issues with Flickholdr’s interface to generate images, but other than that it works like a charm.

Here are some examples of how to use it. Again, simply put this in the src parameter of your img tag:


http://flickholdr.com/200/300


http://flickholdr.com/200/300/sea,sun


http://flickholdr.com/200/300/sea,sun/bw


http://flickholdr.com/200/300/sea,sun/1

Placehold.it

http://placehold.it/

And finally… placehold.it. It’s a much simpler version compared to the last two tools, it’s a solid color image instead of a picture. You can use it when you just need to mark a place as an image placeholder, or if you are just doing a website’s wireframe, etc.

Here are some examples of how to use it. Again, simply put this in the src parameter of your img tag:


http://placehold.it/350x150


http://placehold.it/300x100/000/fff/&text=Custom%20Text


http://placehold.it/600x200/000/fff.png

  • By the number people
  • Gut reaction people

Approaching by-the-number clients

Calculate how much money their wasting by calculating the inefficiency of the current system. How much time employees loose because of  poor navigation and lack of design standards.

Tools for geting quantitative data:

  • Usability Testing — Test your site with 5 people, fix the major issues and test again
  • Google Analytics — Site metrics (http://google.com/analytics)
  • Crazy Egg — Heatmap application (http://www.crazyegg.com/)
  • Userfly — Record and replay interactions made by your visitors (http://userfly.com/)
  • Feedback Army — $15 for 10 remote usability test (http://www.feedbackarmy.com/)
  • 5 second test — How your page to testers for 5 seconds and get their feedback (http://fivesecondtest.com/)
  • User Testing — http://www.usertesting.com/
  • ClickTale — Heatmap application (http://www.clicktale.com/)

Approaching gut-reaction clients

Tell them a good story, show them how their site can go bad, worst case scenarios, make them feel like their missing something out!

Tools for Information Architecture

  • Bubbl.us — Mapping tool to layout site structure (https://bubbl.us/)
  • Axure — Dynamic visualizations, details and diagram of flows (http://www.axure.com/)
  • HotGloo — Wireframing tool (http://www.hotgloo.com/)

Where to get started?

  • Check for free community templates online
  • State the goals of the website, what is this site going to accomplish
  • Structure the sitemap
  • Wireframe the home page with the primary navigation, content areas and call to actions

WordPress Solutions & Plugins

  • Related posts plugin — show related posts
  • Google custom search — nasty, google formated page
  • Search Everything — select what you can
  • Search suggest — real time search suggest
  • Breadcrumb NavXT — simple breadcrumbs plugin
  • Smart 404 — it returns content that might be related to the non existing page
  • Dagon Design Sitemap Generator — Sitemap generator
